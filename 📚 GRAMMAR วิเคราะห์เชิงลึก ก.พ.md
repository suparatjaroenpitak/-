## วิธีอ่านโจทย์ Grammar ให้เร็ว

| ขั้น | ต้องดูอะไร | ถ้าเจอแบบนี้ให้คิดว่า |
|---|---|---|
| 1 | ดูตัวเลือกก่อน | ตัวเลือกบอกหัวข้อสอบได้เร็วที่สุด เช่น `who/which/where` = Relative Pronoun |
| 2 | ดูคำหน้าและคำหลังช่องว่าง | ตำแหน่งของ blank บอกชนิดคำ เช่น `___ + noun` มักเป็น adjective |

| 5 | ใช้ความหมายยืนยัน | หลังเหลือ 1-2 ข้อ ค่อยแปลเพื่อเลือกคำที่เข้าบริบทที่สุด |

**สูตรจำหลัก:**  
ข้อสอบ Grammar ส่วนใหญ่ไม่ต้องแปลทั้งประโยคก่อน ให้เริ่มจาก **Choice -> Blank Position -> Keyword -> Structure -> Meaning**

| `as`, `than`, `the most`, `more`, `-er` | Comparison | `as...as`, `than`, `the + superlative` |

# 1. Relative Pronouns

## Topic Identification

รู้ว่าเป็นเรื่อง Relative Pronoun เมื่อ choice เป็นกลุ่มคำเชื่อมขยายคำนาม เช่น `who`, `whom`, `which`, `that`, `where`, `whose` และประโยคมีคำนามอยู่หน้าช่องว่าง

ตัวอย่าง:

> The woman ___ lives next door is a nurse.

> This is the school ___ I studied.

## Clue Spotting

| Clue ในโจทย์ | ชี้ไปที่ |
|---|---|
| noun หน้า blank เป็นคน และหลัง blank เป็น verb | `who` |
| noun หน้า blank เป็นคน และหลัง blank มี subject + verb แล้ว | `whom` |
| noun หน้า blank เป็นสิ่งของ/สัตว์ | `which` หรือ `that` |
| noun หน้า blank เป็นสถานที่ และหลัง blank เป็นประโยคครบ | `where` |
| หลัง blank มี noun ทันที เช่น `___ car`, `___ bag` | `whose` |

## Choice Analysis

โจทย์ตัวอย่าง:

> The student ___ bag was stolen reported it to the teacher.

| Choice | วิเคราะห์ | ผล |
|---|---|---|
| `who` | ใช้แทนคนที่เป็นประธาน แต่หลังช่องว่างมี `bag` เป็น noun ทันที | ผิด |
| `whom` | ใช้แทนคนที่เป็นกรรม ไม่ได้บอกความเป็นเจ้าของ | ผิด |
| `which` | ใช้แทนสิ่งของ ไม่เหมาะกับ `student` | ผิด |
| `whose` | แปลว่า "ซึ่ง...ของเขา/เธอ" ใช้หน้า noun ได้พอดี: `whose bag` | ถูก |

## Pattern Recognition

| Pattern | คำตอบที่มักใช้ |
|---|---|
| คน + ___ + verb | `who` |
| คน + ___ + subject + verb | `whom` |
| สิ่งของ + ___ + verb | `which` / `that` |
| สถานที่ + ___ + subject + verb | `where` |
| คน/สิ่งของ + ___ + noun | `whose` |

## Instant Kill

**หลังช่องว่างเป็น noun ทันที -> ตอบ `whose` ก่อนคิดเรื่องอื่น**  
เช่น `___ name`, `___ car`, `___ responsibility`, `___ office`

# 2. Parts of Speech & Word Forms

## Topic Identification

ถ้าตัวเลือกเป็นคำรากเดียวกันหลายรูป เช่น noun, verb, adjective, adverb แปลว่าโจทย์วัด **Word Form / Part of Speech**

ตัวอย่าง choice:

| Choice | ชนิดคำ |
|---|---|
| `care` | noun / verb |
| `careful` | adjective |
| `carefully` | adverb |
| `careless` | adjective ความหมายลบ |

## Clue Spotting

| ตำแหน่ง blank | ต้องการคำชนิดใด |
|---|---|
| `a/an/the + ___ + noun` | adjective |
| `verb + ___` | adverb |
| `be/linking verb + ___` | adjective |
| `___ + verb` | noun / pronoun เป็นประธาน |
| `preposition + ___` | noun หรือ gerund |

## Choice Analysis

โจทย์ตัวอย่าง:

> She answered the question very ___.

| Choice | วิเคราะห์ | ผล |
|---|---|---|
| `careful` | adjective ใช้ขยาย noun หรืออยู่หลัง be/linking verb | ผิด |
| `carefully` | adverb ใช้ขยาย verb `answered` และมี `very` ขยายได้ | ถูก |
| `care` | noun/verb ไม่เข้าหลัง `very` ในตำแหน่งนี้ | ผิด |
| `careless` | adjective และความหมายไม่เข้ากับตำแหน่ง | ผิด |

## Pattern Recognition

| โครงสร้างรอบช่องว่าง | คำที่ต้องเติม |
|---|---|
| `She is ___` | adjective |
| `She works ___` | adverb |
| `Her ___ impressed us` | noun |
| `a ___ employee` | adjective |
| `without ___` | noun / V.ing |

## Instant Kill

**หลัง verb ทั่วไป ถ้า choice มีคำลงท้าย `-ly` ให้เล็ง adverb ก่อน**  
แต่ต้องระวังคำอย่าง `friendly`, `lovely`, `early` ที่ไม่ได้เป็น adverb แบบปกติเสมอไป

# 3. Tenses

## Topic Identification

รู้ว่าเป็น tense เมื่อ choice เป็นกริยาตัวเดียวกันแต่ต่างรูป เช่น `work`, `works`, `worked`, `has worked`, `will work`

## Clue Spotting

| Keyword | Tense ที่มักใช้ |
|---|---|
| `yesterday`, `last...`, `...ago` | Past Simple |
| `now`, `at the moment`, `Look!` | Present Continuous |
| `since`, `for`, `already`, `yet`, `just` | Present Perfect |
| `tomorrow`, `next...`, `soon` | Future |
| `every day`, `usually`, `often`, `always` | Present Simple |
| `when + past`, `while + past continuous` | Past Simple / Past Continuous |

## Choice Analysis

โจทย์ตัวอย่าง:

> I ___ here since 2020.

| Choice | วิเคราะห์ | ผล |
|---|---|---|
| `work` | Present Simple ไม่พอ เพราะมี `since` บอกช่วงเริ่มจากอดีตถึงปัจจุบัน | ผิด |
| `worked` | Past Simple จบในอดีต ไม่เน้นต่อถึงปัจจุบัน | ผิด |
| `have worked` | Present Perfect ใช้กับ `since 2020` ได้ตรง | ถูก |
| `will work` | Future ไม่เข้ากับ `since` | ผิด |

## Pattern Recognition

| Pattern | คำตอบ |
|---|---|
| `since/for + เวลา` | `has/have + V.3` |
| `last/yesterday/ago` | `V.2` |
| `every/often/usually` | `V.1` หรือ `V.s/es` |
| `now/at the moment` | `is/am/are + V.ing` |
| `by the time + past` | มักใช้ Past Perfect ในอีก clause |

## Instant Kill

**เจอ `since` หรือ `for` แล้ว choice มี `has/have + V.3` ให้เลือกไว้ก่อน**  
โดยเช็ก subject ให้ตรง: `He has`, `They have`

# 4. Prepositions

## Topic Identification

ถ้า choice เป็น `in`, `on`, `at`, `for`, `since`, `by`, `with`, `about`, `from`, `to` มักวัด preposition

## Clue Spotting

| Clue | คำที่มักใช้ |
|---|---|
| เวลาเป็นจุด เช่น `7 o'clock`, `noon`, `night` | `at` |
| วัน/วันที่ เช่น `Monday`, `June 21` | `on` |
| เดือน/ปี/ช่วงยาว เช่น `June`, `2026`, `the morning` | `in` |
| ระยะเวลา เช่น `two years` | `for` |
| จุดเริ่มเวลา เช่น `2020`, `Monday` | `since` |
| deadline เช่น `Friday`, `5 p.m.` | `by` |

## Choice Analysis

โจทย์ตัวอย่าง:

> The meeting will start ___ 9 a.m.

| Choice | วิเคราะห์ | ผล |
|---|---|---|
| `in` | ใช้กับเดือน ปี ช่วงเวลาใหญ่ | ผิด |
| `on` | ใช้กับวัน/วันที่ | ผิด |
| `at` | ใช้กับเวลาระบุเป็นจุด | ถูก |
| `for` | ใช้กับระยะเวลา ไม่ใช่เวลาเริ่ม | ผิด |

## Pattern Recognition

| Pattern | คำตอบ |
|---|---|
| `interested ___` | `in` |
| `good ___` | `at` |
| `responsible ___` | `for` |
| `depend ___` | `on` |
| `different ___` | `from` |
| `apply ___ a job` | `for` |

## Instant Kill

**เวลาเป็นจุดเล็ก -> `at`, วันเป็นแผ่น -> `on`, เดือน/ปีเป็นกรอบใหญ่ -> `in`**  
จำเป็นภาพ: `at 7`, `on Monday`, `in 2026`

# 5. Subject-Verb Agreement

## Topic Identification

ถ้า choice เป็นกริยารูปเอกพจน์/พหูพจน์ เช่น `is/are`, `has/have`, `do/does`, `go/goes` แปลว่าโจทย์อาจวัดความสอดคล้องระหว่างประธานกับกริยา

## Clue Spotting

| Clue | วิธีคิด |
|---|---|
| มี phrase คั่นกลาง เช่น `with`, `along with`, `together with` | ตัด phrase คั่นกลางออกก่อน |
| ประธานขึ้นต้นด้วย `Each`, `Every`, `Either`, `Neither` | มักเป็นเอกพจน์ |
| ประธานเป็น `A number of + plural noun` | ใช้กริยาพหูพจน์ |
| ประธานเป็น `The number of + plural noun` | ใช้กริยาเอกพจน์ |
| มี `and` เชื่อมประธาน | มักเป็นพหูพจน์ |

## Choice Analysis

โจทย์ตัวอย่าง:

> The manager, along with his assistants, ___ attending the meeting.

| Choice | วิเคราะห์ | ผล |
|---|---|---|
| `are` | โดน `assistants` หลอก แต่ subject จริงคือ `The manager` | ผิด |
| `is` | subject จริงเป็นเอกพจน์ ใช้ `is` | ถูก |
| `were` | tense อาจไม่ตรง และเป็นพหูพจน์ | ผิด |
| `have` | ไม่เข้ากับโครง `___ attending` | ผิด |

## Pattern Recognition

| Pattern | กริยา |
|---|---|
| `S1, with/together with/along with + S2, ___` | ตาม S1 |
| `Each/Every + singular noun ___` | singular verb |
| `Neither/Either of + plural noun ___` | singular verb |
| `A number of + plural noun ___` | plural verb |
| `The number of + plural noun ___` | singular verb |

## Instant Kill

**ขีดฆ่าคำคั่นกลางระหว่าง subject กับ verb ก่อนเลือกคำตอบ**  
เช่น `The president, along with his staff, ___` ให้ดูแค่ `The president ___`

# 6. Passive Voice

## Topic Identification

ตัวเลือกมักปน active/passive เช่น `build`, `built`, `is built`, `was building`, `has been built` ให้ดูว่าประธาน “ทำเอง” หรือ “ถูกกระทำ”

## Clue Spotting

| Clue | ชี้ว่าเป็น Passive |
|---|---|
| ประธานเป็นสิ่งของ/เอกสาร/อาคาร/งาน | มักถูกกระทำ |
| มี `by + ผู้กระทำ` | โอกาสสูงเป็น passive |
| ความหมายคือ "ถูก..." | ใช้ `be + V.3` |
| มี keyword เวลา | ช่วยเลือก tense ของ `be` |

## Choice Analysis

โจทย์ตัวอย่าง:

> The report ___ by the committee yesterday.

| Choice | วิเคราะห์ | ผล |
|---|---|---|
| `approved` | ไม่มี `be` ถ้าใช้เป็น passive จะขาดโครงสร้าง | ผิด |
| `was approved` | Past Simple Passive เพราะมี `yesterday` และ `by the committee` | ถูก |
| `is approving` | active continuous แปลว่ารายงานกำลังอนุมัติเอง | ผิด |
| `has approved` | active และประธานทำเองไม่ได้ | ผิด |

## Pattern Recognition

| Tense | Passive Form |
|---|---|
| Present Simple | `is/am/are + V.3` |
| Past Simple | `was/were + V.3` |
| Present Perfect | `has/have been + V.3` |
| Future | `will be + V.3` |
| Modal | `modal + be + V.3` |

## Instant Kill

**เจอ `by + คน/หน่วยงาน` และประธานถูกกระทำ -> หา `be + V.3` ทันที**

# 7. Conjunctions & Connectors

## Topic Identification

ถ้า choice เป็นคำเชื่อม เช่น `because`, `although`, `so`, `therefore`, `if`, `unless`, `while`, `when` โจทย์วัดความสัมพันธ์ของความหมายระหว่างสองส่วน

## Clue Spotting

| ความสัมพันธ์ | Keyword / Connector |
|---|---|
| เหตุผล | `because`, `since`, `as` |
| ผลลัพธ์ | `so`, `therefore`, `as a result` |
| ขัดแย้ง | `although`, `though`, `but`, `however` |
| เงื่อนไข | `if`, `unless` |
| เวลา | `when`, `while`, `before`, `after` |

## Choice Analysis

โจทย์ตัวอย่าง:

> ___ it was raining, we went out for dinner.

| Choice | วิเคราะห์ | ผล |
|---|---|---|
| `Because` | ถ้าเพราะฝนตกจึงออกไปกินข้าว ความหมายแปลก | ผิด |
| `Although` | แม้ฝนตก แต่ก็ออกไปข้างนอก เป็นความขัดแย้ง | ถูก |
| `So` | `so` มักอยู่กลางเชื่อมผล ไม่วางต้นประโยคแบบนี้ในข้อสอบพื้นฐาน | ผิด |
| `If` | เงื่อนไขไม่ตรงกับความหมาย | ผิด |

## Pattern Recognition

| Pattern | คำตอบ |
|---|---|
| เหตุ + ผล | `because` / `since` |
| ข้อเท็จจริงขัดกัน | `although` / `but` / `however` |
| เงื่อนไข | `if` |
| เงื่อนไขเชิงลบ | `unless` |
| ผลลัพธ์จากประโยคก่อน | `therefore` / `so` |

## Instant Kill

**ถ้าสองประโยคให้ความหมายสวนกัน ให้หา `although / but / however` ก่อน**  
โดยดูว่าช่องว่างอยู่ตำแหน่งไหน: หน้า clause ใช้ `although`, กลางประโยคใช้ `but`, ขึ้นประโยคใหม่ใช้ `however`

# 8. Infinitive & Gerund

## Topic Identification

Choice มักเป็นกริยาตัวเดียวกันหลายรูป เช่น `go`, `to go`, `going`, `gone` แปลว่าโจทย์วัด verb pattern

## Clue Spotting

| คำหน้า blank | รูปที่ตามหลัง |
|---|---|
| modal: `can`, `must`, `should`, `may` | `V.1` |
| `want`, `decide`, `plan`, `hope` | `to + V.1` |
| `enjoy`, `avoid`, `finish`, `mind`, `consider` | `V.ing` |
| preposition เช่น `of`, `for`, `without`, `before` | `V.ing` |
| causative `make`, `let`, `help` | `V.1` |

## Choice Analysis

โจทย์ตัวอย่าง:

> She enjoys ___ English every morning.

| Choice | วิเคราะห์ | ผล |
|---|---|---|
| `study` | หลัง `enjoy` ต้องเป็น gerund | ผิด |
| `to study` | `enjoy` ไม่ตามด้วย infinitive ใน pattern มาตรฐาน | ผิด |
| `studying` | `enjoy + V.ing` | ถูก |
| `studied` | V.2/V.3 ไม่เข้าตำแหน่ง | ผิด |

## Pattern Recognition

| Pattern | ตัวอย่าง |
|---|---|
| `want + to V.1` | `want to learn` |
| `avoid + V.ing` | `avoid making mistakes` |
| `without + V.ing` | `without asking` |
| `should + V.1` | `should study` |
| `make + object + V.1` | `make him wait` |

## Instant Kill

**หลัง preposition ถ้าต้องใช้ verb ให้เปลี่ยนเป็น `V.ing` ทันที**  
เช่น `before leaving`, `after finishing`, `without telling`, `interested in learning`

# 9. Pronouns & Quantifiers

## Topic Identification

ถ้า choice เป็น `he/him/his`, `they/them/their`, `some/any`, `much/many`, `few/little` โจทย์วัด pronoun หรือ quantifier

## Clue Spotting

| Clue | วิธีเลือก |
|---|---|
| ช่องว่างเป็นประธาน | ใช้ subject pronoun: `he`, `she`, `they`, `we` |
| ช่องว่างเป็นกรรม | ใช้ object pronoun: `him`, `her`, `them`, `us` |
| หน้า noun แสดงเจ้าของ | ใช้ possessive adjective: `his`, `her`, `their` |
| noun นับได้พหูพจน์ | ใช้ `many`, `few`, `a few` |
| noun นับไม่ได้ | ใช้ `much`, `little`, `a little` |
| ประโยคปฏิเสธ/คำถาม | มักใช้ `any` |

## Choice Analysis

โจทย์ตัวอย่าง:

> There isn't ___ water in the bottle.

| Choice | วิเคราะห์ | ผล |
|---|---|---|
| `many` | ใช้กับ noun นับได้พหูพจน์ | ผิด |
| `much` | ใช้กับ noun นับไม่ได้ และเข้ากับประโยคปฏิเสธ | ถูก |
| `few` | ใช้กับ noun นับได้พหูพจน์ | ผิด |
| `a few` | ใช้กับ noun นับได้พหูพจน์ และความหมายไม่ตรง | ผิด |

## Pattern Recognition

| Pattern | คำตอบ |
|---|---|
| `___ is my friend` | subject pronoun |
| `I saw ___ yesterday` | object pronoun |
| `___ book is on the table` | possessive adjective |
| `many/few + plural count noun` | `many books`, `few people` |
| `much/little + uncountable noun` | `much money`, `little information` |

## Instant Kill

**ดู noun หลัง quantifier ก่อนเสมอ:**  
`many/few` ไปกับนับได้พหูพจน์, `much/little` ไปกับนับไม่ได้

# 10. If-Clause & Wish

## Topic Identification

เห็น `if`, `unless`, `wish`, หรือ choice มี `will/would`, `were`, `had V.3`, `would have V.3` ให้คิดเรื่องเงื่อนไขและความจริงตรงข้าม

## Clue Spotting

| Pattern | ความหมาย |
|---|---|
| `If + Present Simple, will + V.1` | เงื่อนไขจริงในอนาคต |
| `If + Past Simple, would + V.1` | สมมติปัจจุบัน/อนาคต |
| `If + Past Perfect, would have + V.3` | สมมติอดีต |
| `wish + Past Simple` | อยากให้ปัจจุบันต่างจากจริง |
| `wish + Past Perfect` | เสียดายในอดีต |

## Choice Analysis

โจทย์ตัวอย่าง:

> If I had studied harder, I ___ the exam.

| Choice | วิเคราะห์ | ผล |
|---|---|---|
| `will pass` | ใช้กับ If Type 1 ไม่ใช่ `had studied` | ผิด |
| `would pass` | ใช้กับ If Type 2 | ผิด |
| `would have passed` | If Type 3: `If + had V.3, would have V.3` | ถูก |
| `pass` | โครงสร้างไม่ครบ | ผิด |

## Pattern Recognition

| If-Clause | Main Clause |
|---|---|
| `If + V.1/s/es` | `will + V.1` |
| `If + V.2` | `would + V.1` |
| `If + had + V.3` | `would have + V.3` |
| `Unless + V.1/s/es` | `will + V.1` |
| `I wish + V.2` | ปัจจุบันตรงข้ามจริง |

## Instant Kill

**เห็น `had + V.3` ใน if-clause -> หา `would have + V.3` ใน main clause ทันที**

# 11. Comparison

## Topic Identification

ถ้า choice มี `as`, `than`, `more`, `most`, `better`, `best`, `-er`, `-est` แปลว่าโจทย์วัด comparison

## Clue Spotting

| Clue | คำตอบที่มักใช้ |
|---|---|
| `as ___ as` | adjective/adverb รูปปกติ |
| `___ than` | comparative: `-er` หรือ `more` |
| `the ___` | superlative: `-est` หรือ `most` |
| `one of the ___` | superlative + plural noun |
| `much/even/far ___ than` | comparative |

## Choice Analysis

โจทย์ตัวอย่าง:

> This test is ___ than the previous one.

| Choice | วิเคราะห์ | ผล |
|---|---|---|
| `easy` | รูปปกติ ใช้กับ `as...as` มากกว่า | ผิด |
| `easier` | มี `than` ต้องใช้ comparative | ถูก |
| `easiest` | ขั้นสุดต้องมี `the` และเทียบมากกว่า 2 สิ่ง | ผิด |
| `as easy` | ต้องครบ `as easy as` | ผิด |

## Pattern Recognition

| Pattern | ตัวอย่างคำตอบ |
|---|---|
| `as ___ as` | `as tall as` |
| `___ than` | `taller than`, `more useful than` |
| `the ___ in/of` | `the tallest`, `the most useful` |
| `one of the ___ + plural noun` | `one of the best students` |
| `less ___ than` | `less expensive than` |

## Instant Kill

**เจอ `than` -> ใช้ขั้นกว่า, เจอ `the + blank + in/of` -> ใช้ขั้นสุด**

## ตารางสรุปสูตรลัด 5 วินาที

| Clue ที่เห็นทันที | ตอบ/คิดทันที |
|---|---|
| `since/for + เวลา` | Present Perfect: `has/have + V.3` |
| `yesterday/last/ago` | Past Simple: `V.2` |
| `by + ผู้กระทำ` | Passive: `be + V.3` |
| `than` | Comparative |
| `as ___ as` | รูปปกติ ไม่ใช้ขั้นกว่า |
| หลัง modal | `V.1` |
| หลัง preposition แล้วเป็น verb | `V.ing` |
| หลัง blank เป็น noun ทันทีใน relative clause | `whose` |
| `___ + noun` และ choice เป็น word form | adjective |
| `verb + ___` และ choice เป็น word form | adverb |
| `Each/Every + singular noun` | singular verb |
| `The number of + plural noun` | singular verb |
| `A number of + plural noun` | plural verb |
| `If + had V.3` | `would have + V.3` |
| noun นับไม่ได้ | `much/little` |
| noun นับได้พหูพจน์ | `many/few` |
